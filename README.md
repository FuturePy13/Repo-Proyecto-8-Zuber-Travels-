**Análisis de Datos para Zuber: Comportamiento de Pasajeros y Efecto del Clima en la Demanda**

**Objetivo del proyecto**
Analizar los datos de una empresa de ride-sharing (Zuber), identificar patrones de uso, evaluar el impacto del clima en la demanda y obtener insights estratégicos para competir en el mercado de Chicago.

**Tecnologías utilizadas**

* Python

* Pandas

* NumPy

* Matplotlib

* Stats Library / SciPy (hipótesis y variaciones)

**Metodología de trabajo**
1. Limpieza y preparación de datos

* Normalización de columnas y tipos de datos

* Eliminación de duplicados y nulos

* Junta y estructuración de tablas mediante consultas SQL

* Preparación del dataset para análisis temporal y climático

2. Análisis Exploratorio (EDA)

* Patrones de viaje por hora, día de la semana y temporada

* Distribución de viajes por zonas y picos de demanda

* Análisis del comportamiento de los usuarios

* Gráficos con Matplotlib para visualizar tendencias clave

3. Integración de datos climáticos

* Relación entre precipitaciones, temperatura y velocidad del viento con frecuencia de viajes

* Comparación entre días con clima normal y condiciones climáticas extremas

4. Análisis competitivo

* Revisión de datos de empresas competidoras

* Identificación de patrones de mercado y oportunidades para Zuber

5. Validación estadística

**Prueba de hipótesis:**
“Las condiciones climáticas afectan la frecuencia de los viajes.”

**Resultados e insights clave**
Se encontraron patrones temporales fuertes: mayor demanda en horas pico y fines de semana.

* El clima demostró tener un impacto notable en la demanda: la frecuencia de viajes cae o sube significativamente según condiciones como lluvia o temperaturas extremas.

* Los competidores presentan actividad más alta en ciertos distritos, mostrando oportunidades de posicionamiento para Zuber.

* La hipótesis climática fue confirmada estadísticamente, generando insights valiosos para la planificación operativa.
